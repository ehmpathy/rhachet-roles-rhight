# 2.1.criteria.blackbox

blackbox criteria for the patenter role — what experience must be delivered.

---

## episode experience

### usecase.1 = prior art research

the inventor wants to understand the patent landscape before investing time in a full application.

```
given(inventor has an idea for "machine learning model compression")
  when(they search for prior art)
    then(they receive a list of relevant patents)
      sothat(they can see what already extant)
    then(each result includes title, abstract, and relevance score)
      sothat(they can prioritize which patents to read)
    then(results are sorted by relevance)
      sothat(most relevant patents appear first)

  when(they fetch a specific patent by exid)
    then(they receive the full patent text)
      sothat(they can read the detailed claims)
    then(they receive the claims section separately)
      sothat(they can focus on what the patent actually protects)
    then(they receive figure references if available)
      sothat(they can understand the diagrams)

given(inventor searches with a vague query like "AI")
  when(they search for prior art)
    then(they receive results but with a alert about query specificity)
      sothat(they know to refine their search)

given(inventor searches for something with no matches)
  when(they search for prior art)
    then(they receive an empty result set)
      sothat(they know the search found no results)
    then(they receive suggestions for query refinement)
      sothat(they can try alternative phrasings)
```

### usecase.2 = patent proposal route

the inventor wants to go through a structured process to analyze patentability and prepare materials.

```
given(inventor wants to propose a patent)
  when(they run patent.propose)
    then(a route directory is created with all stone files)
      sothat(they have a structured workspace)
    then(the 0.idea.md file is created with a template)
      sothat(they know what information to provide)
    then(stdout shows the route structure with mascot header)
      sothat(they see what phases they'll go through)
    then(the route is bound to their current branch)
      sothat(progress is tracked via hooks)

  when(they run patent.propose --open nvim)
    then(nvim opens with 0.idea.md)
      sothat(they can start writing immediately)

  when(they run patent.propose --open code)
    then(vscode opens with 0.idea.md)
      sothat(they can start writing in their preferred editor)

given(inventor has filled out 0.idea.md)
  when(they proceed through the research phase)
    then(prior art is searched automatically based on their idea description)
      sothat(they don't have to manually craft queries)
    then(favorable prior art is categorized in 3.1.research.prior-art.favorable.stone)
      sothat(they see patents that support their novelty argument)
    then(adverse prior art is categorized in 3.1.research.prior-art.adverse.stone)
      sothat(they see patents that may challenge their claims)

given(inventor has completed prior art research)
  when(they proceed through the distillation phase)
    then(claims are drafted based on what's novel vs what's covered)
      sothat(they understand their patentable space)
    then(potential office action objections are anticipated)
      sothat(they can prepare responses in advance)

given(inventor has completed distillation)
  when(they proceed through blueprint and delivery)
    then(a latex-formatted patent application is produced)
      sothat(they have materials ready for attorney review)
```

### usecase.3 = iterative refinement

the inventor wants to refine their search and claims as they learn more.

```
given(inventor has searched once and found relevant prior art)
  when(they search again with refined terms)
    then(they receive updated results)
      sothat(they can explore the landscape iteratively)

given(inventor discovers new prior art during the route)
  when(they re-run the research phase)
    then(the prior art categorization is updated)
      sothat(their analysis stays current)
```

## exchange experience

### exchange.1 = patent.priors.search

```
input:
  --query <string>    # natural language description of the invention

output (success):
  - list of patent results
  - each result contains:
    - exid: external identifier (e.g., US20210234567A1)
    - title: patent title
    - abstract: patent abstract
    - relevance: score indicating match quality
  - sorted by relevance descending

output (no results):
  - empty list
  - suggestion for query refinement

errors:
  - invalid query (empty or too short)
  - API unavailable
  - rate limit exceeded
```

### exchange.2 = patent.priors.fetch

```
input:
  --exid <string>       # application number (8 digits, e.g., 19394030)
  --cache skip          # optional: bypass cache and force fresh API call

output (success):
  - full patent document with:
    - title
    - abstract (full text from document XML)
    - claims (numbered list, full text from document XML)
    - description (full specification text from document XML)
    - figures (downloaded image files)
  - prosecution history with:
    - timeline events (codes, descriptions, dates from eventDataBag)
    - **FULL PROSECUTION DOCUMENTS** (REQUIRED):
      - office action PDFs (rejections, requirements)
      - applicant responses (arguments, amendments)
      - notices (allowance, abandonment)
      - all other correspondence with USPTO
  - metadata:
    - date filed
    - date published
    - inventors
    - assignee
    - patent number (if granted)

**CRITICAL: prosecution history requires BOTH:**
1. timeline events (what happened and when)
2. **actual document content** (the PDFs — office actions, responses, amendments)

timeline events alone are INSUFFICIENT. the whole point is to read what the examiner said and how the applicant responded.

output (not found):
  - error: patent not found
  - suggestion to verify exid format

errors:
  - invalid exid format
  - patent not found
  - API unavailable
  - document XML unavailable (metadata exists but document not yet published)

fetch phases:
  1. metadata fetch: GET /api/v1/patent/applications/{exid}
     - returns prosecution timeline events (eventDataBag), application dates, grant info
     - contains grantDocumentMetaData.fileLocationURI for full granted patent document
  2. document fetch: GET {fileLocationURI}
     - returns full patent XML with claims, description, abstract
     - extract figure references from XML
  3. figure fetch: GET each figure URI from document
     - downloads actual image files (PNG, TIFF, etc.)
  4. **prosecution documents fetch (REQUIRED):**
     - GET /api/v1/patent/applications/{exid}/documents
     - returns list of all prosecution documents with download URIs
     - for each document:
       - GET the document content (PDF, typically)
       - cache to .cache/patents/{exid}.docs/{docCode}.{date}.pdf
     - document types include:
       - office actions (CTFR, CTNF, etc.)
       - applicant responses (A..., RESP, etc.)
       - notices (MN/=., NOA, etc.)
       - amendments, claims, arguments

cache structure:
  - .cache/patents/{exid}.meta.json         # raw metadata API response (timeline events)
  - .cache/patents/{exid}.doc.xml           # full granted patent document XML
  - .cache/patents/{exid}.fig-{n}.{ext}     # figure images (fig-1.png, fig-2.tiff, etc.)
  - .cache/patents/{exid}.docs/             # prosecution documents directory (REQUIRED)
    - {docCode}.{date}.pdf                  # e.g., CTFR.2019-03-15.pdf (final rejection)
    - {docCode}.{date}.pdf                  # e.g., A....2019-06-10.pdf (response)
    - ...all prosecution correspondence

cache rules:
  - always cache raw responses (never parse/transform for cache)
  - --cache skip bypasses all cached files and forces fresh fetch
  - patent data is immutable, so cache never stales
  - if document XML unavailable, emit metadata + prosecution docs
  - **prosecution documents are REQUIRED** — timeline events alone are insufficient
```

### exchange.3 = patent.propose

```
input:
  (no required args)
  --open <opener>    # optional: nvim, code, etc.

output (success):
  - route directory created at .route/v{date}.patent.propose/
  - files created:
    - 0.idea.md (template for inventor to fill)
    - 1.vision.stone
    - 3.1.research.prior-art.favorable.stone
    - 3.1.research.prior-art.adverse.stone
    - 3.2.distill.claims.prior-art.stone
    - 3.2.distill.claims.patentable.stone
    - 3.2.distill.strategy.officeactions.stone
    - 3.3.blueprint.patent.stone
    - 5.1.deliver.patent.latex.stone
  - stdout: mascot header + route tree + branch bound info
  - if --open specified: editor opens with 0.idea.md

errors:
  - route already extant for this branch
  - git not initialized
  - editor not found (if --open specified)
```

## boundary conditions

### query boundaries

| condition | behavior |
|-----------|----------|
| empty query | error: query required |
| query < 3 chars | error: query too short |
| query > 1000 chars | error: query too long |
| query with special chars | sanitized, search proceeds |

### exid boundaries

| condition | behavior |
|-----------|----------|
| valid USPTO format (e.g., US20210234567A1) | fetch proceeds |
| valid EPO format (future) | error: EPO not yet supported |
| invalid format | error: invalid exid format |
| valid format but not found | error: patent not found |

### route boundaries

| condition | behavior |
|-----------|----------|
| no git repo | error: must be in git repository |
| route already extant | error: route already extant, use it or delete |
| branch not feature branch | alert: typically run on feature branches |
| editor not installed | error: editor not found |
