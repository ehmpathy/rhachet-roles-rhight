# 2.3.criteria.blueprint

blueprint criteria for the patenter role — mechanism bounds (not implementation details).

---

## subcomponent contracts

### patent.priors.search

```
contract:
  input:
    query: string (3-1000 chars, sanitized)
  output:
    results: Array<{
      exid: string      # USPTO format
      title: string
      abstract: string
      relevance: number # 0-1 score
    }>
    alert?: string      # if query too vague
    suggestion?: string # if no results
  errors:
    QueryTooShort | QueryTooLong | ApiUnavailable | RateLimitExceeded

guarantees:
  - results sorted by relevance descending
  - relevance scores are comparable across queries
  - API failures surface as typed errors, not silent failures
```

### patent.priors.fetch

```
contract:
  input:
    exid: string (USPTO format validated)
  output:
    patent: {
      title: string
      abstract: string
      claims: Array<{ number: number, text: string }>
      description: string
      figures: Array<{ ref: string, description?: string }>
      prosecutionHistory?: Array<{ date: string, action: string }>
    }
    metadata: {
      dateFiled: string
      datePublished: string
      inventors: string[]
      assignee: string
    }
  errors:
    InvalidExidFormat | PatentNotFound | ApiUnavailable

guarantees:
  - claims are numbered and complete
  - prosecution history included when API provides it
  - figure references extracted even if images not available
```

### patent.propose

```
contract:
  input:
    open?: string (nvim | code | other opener)
  output:
    routePath: string   # .route/v{date}.patent.propose/
    files: string[]     # list of created stone files
    boundBranch: string # current git branch
  stdout:
    mascot header (🦅 eagle)
    route tree with checkmarks
    branch ↔ route bound confirmation
  errors:
    RouteAlreadyExtant | NotGitRepository | EditorNotFound

guarantees:
  - route directory created atomically (all files or none)
  - branch ↔ route bound is persistent across sessions
  - 0.idea.md contains structured template with guided prompts
```

## composition boundaries

### route phase dependencies

```
0.idea.md
  └─► 3.1.research.prior-art.*.stone
        └─► 3.2.distill.claims.*.stone
              └─► 3.3.blueprint.patent.stone
                    └─► 5.1.deliver.patent.latex.stone
```

phase guards enforce this order — later phases cannot complete until prior phases are done.

### data flow between stones

| from | to | data |
|------|-----|------|
| 0.idea.md | 3.1.research | invention description, keywords |
| 3.1.research.favorable | 3.2.distill.claims | patents that support novelty |
| 3.1.research.adverse | 3.2.distill.claims | patents that challenge claims |
| 3.2.distill.claims.patentable | 3.3.blueprint | draft claims |
| 3.2.distill.strategy | 3.3.blueprint | anticipated objections |
| 3.3.blueprint | 5.1.deliver | structured application |

## integration boundaries

### USPTO API

```
boundary:
  - search endpoint: patent search by keywords
  - fetch endpoint: patent document by exid
  - rate limits: TBD (research needed)
  - authentication: TBD (research needed)

fallback:
  - if USPTO unavailable: surface error with retry guidance
  - if rate limited: surface error with wait time
```

### bhrain route system

```
boundary:
  - route creation: .route/v{date}.{name}/
  - branch bind: .branch/.bind/{branch} ↔ route
  - hooks: drive phases via guard/stone pattern

guarantees:
  - follows bhrain conventions for route structure
  - uses standard mascot output format
  - integrates with extant hook system
```

### editor integration

```
boundary:
  - opener detection: which $EDITOR or explicit --open
  - file open: opener invoked with 0.idea.md path

guarantees:
  - graceful fallback if editor not found
  - does not block if editor invocation fails
```

## non-goals (explicit exclusions)

- **not legal advice** — tool assists research, does not provide legal counsel
- **not international** — USPTO only in v1, EPO/WIPO as future scope
- **not design patents** — utility patents only in v1
- **not provisional applications** — full utility route only in v1
