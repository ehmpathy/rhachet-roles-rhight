# 3.1.3.research.internal.product.code.test._.v1

research on internal test code patterns relevant to the patenter role.

---

## summary

this research catalogs extant test code patterns in the rhachet-roles-rhight repo that the patenter role should leverage. patterns are categorized as [REUSE], [EXTEND], or [REPLACE].

key findings:
1. **BDD structure pattern** — given/when/then with [caseN]/[tN] labeling via test-fns
2. **spawnSync execution pattern** — run shell skills synchronously from TypeScript tests
3. **genTempDir isolation pattern** — create isolated temp directories per test
4. **stdin JSON injection pattern** — test hooks by providing JSON via stdin
5. **snapshot testing pattern** — toMatchSnapshot() with output sanitization
6. **exit code semantics pattern** — 0=success, 1=malfunction, 2=constraint
7. **afterAll cleanup pattern** — ensure temp directories are removed after tests
8. **PATH injection mock pattern** — prepend fake bin directory for CLI mocks
9. **useBeforeAll/useBeforeEach pattern** — share expensive setup across tests
10. **operation function organization** — runSkill(), runHook*() wrappers
11. **env injection pattern** — pass environment variables to spawned processes
12. **output sanitization pattern** — replace timestamps, paths for reproducible snapshots

## pattern.1 = BDD structure

**status**: [REUSE]

**location**: `src/domain.roles/mechanic/skills/declapract.upgrade/declapract.upgrade.integration.test.ts`

**pattern**:
```typescript
import { given, when, then, useBeforeAll } from 'test-fns';

describe('skill-name', () => {
  given('[case1] scenario description', () => {
    const scene = useBeforeAll(async () => {
      // setup
      return { entity };
    });

    when('[t0] action description', () => {
      then('expected outcome', async () => {
        const result = await performAction(scene.entity);
        expect(result).toEqual(expectedValue);
      });
    });
  });
});
```

**reuse for patenter**:
- same given/when/then structure for all tests
- [caseN] labels for given blocks
- [tN] labels for when blocks, reset per given
- useBeforeAll for expensive setup (API calls, temp dir creation)

## pattern.2 = spawnSync execution

**status**: [REUSE]

**location**: `src/domain.roles/mechanic/skills/declapract.upgrade/declapract.upgrade.integration.test.ts`

**pattern**:
```typescript
import { spawnSync } from 'child_process';

const runSkill = (
  args: string[],
  options: { cwd: string; env?: Record<string, string> }
): { stdout: string; stderr: string; exitCode: number } => {
  const result = spawnSync('bash', [scriptPath, ...args], {
    encoding: 'utf-8',
    cwd: options.cwd,
    env: { ...process.env, ...options.env },
    stdio: ['pipe', 'pipe', 'pipe'],
  });
  return {
    stdout: result.stdout ?? '',
    stderr: result.stderr ?? '',
    exitCode: result.status ?? 1,
  };
};
```

**reuse for patenter**:
- same wrapper pattern for patent.priors.search, patent.priors.fetch, patent.propose
- always capture stdout, stderr, exitCode
- always use encoding: 'utf-8' for string output

```typescript
describe('skill-name', () => {
  const tempDirs: string[] = [];

  afterAll(() => {
    tempDirs.forEach((dir) => {
      fs.rmSync(dir, { recursive: true, force: true });
    });
  });

  given('[case1] scenario', () => {
    const scene = useBeforeAll(async () => {
      const tempDir = createTempDir();

      return { tempDir };
    });
  });
});
```

**reuse for patenter**:

- genTempDir handles cleanup automatically

## pattern.8 = PATH injection mock

**status**: [EXTEND]

**location**: documented in briefs, used in various integration tests

**pattern**:
```typescript
const setupMockGh = (tempDir: string): string => {
  const fakeBinDir = path.join(tempDir, '.fakebin');
  fs.mkdirSync(fakeBinDir, { recursive: true });

  fs.writeFileSync(
    path.join(fakeBinDir, 'gh'),
    `#!/bin/bash
if [[ "$1" == "api" ]]; then
  echo '{"items": []}'
  exit 0
fi
exit 1`,
  );
  fs.chmodSync(path.join(fakeBinDir, 'gh'), '755');

  return fakeBinDir;
};

// use in test
const fakeBinDir = setupMockGh(tempDir);
const result = runSkill(args, {
  cwd: tempDir,
  env: { PATH: `${fakeBinDir}:${process.env.PATH}` },
});
```

**extend for patenter**:
- mock USPTO API calls via fake curl/http responses
- mock patent search responses with fixture data
- useful for tests that should not hit real APIs

```typescript
const sanitizeOutput = (stdout: string): string =>
```
