# 3.1.1.research.external.product.access._.v1

research on remote access required to fulfill the patenter role.

---

## summary

the patenter role requires access to patent databases for search and document retrieval. three primary options exist:

1. **USPTO Open Data Portal (ODP)** — official, free, requires API key, rate limited
2. **PatentsView API** — official USPTO, structured data focus, 45 req/min limit
3. **EPO Open Patent Services (OPS)** — european patents, free tier available

recommendation: use USPTO ODP for US patent search and document fetch. use PatentsView for structured analytics queries. avoid scraping Google Patents (no official API, ToS risk).

---

## remote repositories

### USPTO Open Data Portal (ODP)

the primary official source for US patent data.

**contract**:
- search endpoint: `data.uspto.gov/apis/patent-file-wrapper/search`
- documents endpoint: `data.uspto.gov/apis/patent-file-wrapper/documents`
- authentication: API key required (free registration)
- rate limits: TBD (not publicly documented per search results)

> "The new Open Data Portal (ODP) is now live, replacing the Open Data Portal Beta which is scheduled for shutdown on April 20, 2026." [1]

> "You must have an ODP API key to access ODP APIs, and you can obtain one by visiting the ODP Getting Started page." [1]

> "The ODP supports full-text search of the application's full text and allows the simplified search syntax. Additionally, the ODP supports the OpenSearch Query syntax, which is the most powerful search option and allows for complex queries." [7]

**interfaces**: REST API with JSON responses, Swagger documentation available.

### PatentsView API

structured patent analytics API, also official USPTO.

**contract**:
- endpoint: `search.patentsview.org`
- authentication: API key required (X-Api-Key header)
- rate limits: 45 requests/minute per API key
- max results: 100,000 per query
- pagination: cursor-based

> "The API currently allows 45 calls per minute for each API key. If this limit is exceeded the API will return an http status of 429 with a response header Retry-After set to the number of seconds to wait before making subsequent requests." [8]

> "The after option implements Cursor Pagination. If included, the value should be a valid value from the field(s) specified in the sort option." [8]

> "PatentsView is an official API maintained by the USPTO, offering structured access to United States patent data suitable for researchers and developers who want to explore inventor relationships, citation data, patent metadata, and classification systems at scale." [5]

**interfaces**: REST API, JSON responses, extensive query language.

### EPO Open Patent Services (OPS)

for international patent coverage beyond USPTO.

**contract**:
- endpoint: via EPO Developer Portal
- authentication: Consumer Key + Consumer Secret (OAuth)
- rate limits: free tier has data usage limits
- coverage: 130+ million patent documents worldwide

> "To use the EPO OPS 3.2 services, you need to register at the EPO Developer Portal by applying for account registration at https://developers.epo.org/user/register." [10]

> "Espacenet provides free access to over 130 million patent documents. Access to patent data through the EPO Application Programming Interface (API) is available free of charge, though it requires programming knowledge." [5]

**interfaces**: REST API, OAuth authentication.

### Google Patents (NOT RECOMMENDED)

> "Google Patents does not provide a public API, so developers cannot directly access an official Google Patents API." [2]

> "Scraping websites like Google Patents may breach their terms of service, and you should consult legal expertise to address compliance with applicable laws and regulations, and always obtain permission and consider using available official APIs for data access." [2]

third-party scraping services exist (SerpApi, SearchAPI.io, Scrapingdog) but carry legal risk.

## best practices

### authentication

> "The PatentSearch API uses API Keys to authenticate requests, and all API requests must include the header X-Api-Key: {your_key}." [8]

> "Developers need to obtain an API key by registering on the EPO developer portal, and the API key must be included in the request headers for authentication." [10]

**convergence**: all three major APIs (ODP, PatentsView, OPS) require API key registration. none allow anonymous access.

### rate limit handling

> "When limits are exceeded, return a clear error response with HTTP status code 429 (Too Many Requests) and include a Retry-After header indicating when the client can attempt again." [14]

> "Include rate limit details through headers like X-RateLimit-Limit, X-RateLimit-Remaining, and X-RateLimit-Reset to provide users with a transparent view of their usage and remaining quota." [14]

> "Implement exponential backoff with jitter for intelligent retry handling. When an API 429 error lands, it's best to avoid flooding the API with retries. Instead, pause and retry after a short delay and then wait longer with each attempt." [14]

**convergence**: sources [14], [15], [16] all recommend exponential backoff with jitter for rate limit handling.

### caching

> "Implement caching strategies to reduce the number of requests that need to hit the API directly, which can help alleviate some of the load and reduce the likelihood of users hitting rate limits. Use tools like Redis and CDNs to reduce redundant requests and improve performance." [14]

**recommendation**: cache patent documents aggressively (they are immutable after publication). cache search results with shorter TTL.

### bulk data vs API

> "Bulk data refers to putting all static data into a file or set of files, so that all of the data can be acquired with downloads." [17]

> "For programmers and researchers working with large datasets, PatentsView database tables can be bulk downloaded as individual files in a tab-delimited format." [17]

> "For high-volume needs with USPTO data, use bulkdata.uspto.gov to download XML files rather than scraping the search GUI, as it is much faster and less restricted." [19]

**decision**: use API for interactive search and single document fetch. use bulk download for analytics or offline processing.

## client libraries

### patent-client (Python)

> "The patent_client package provides easy access to a number of USPTO and EPO patent data resources, all through a familiar ORM-like interface." [12]

> "All of the models are accessible directly at patent_client. Further, all the API's used in this library are Read Only, so there is no way to add, delete, or modify records." [12]

documentation: https://patent-client.readthedocs.io/

### pyUSPTO (Python)

> "A Modern Python client for accessing the United States Patent and Trademark Office (USPTO) Open Data Portal (ODP) APIs." [13]

### uspto-opendata-python (Python)

> "A client library for accessing the USPTO Open Data APIs, written in Python." [13]

documentation: https://docs.ip-tools.org/uspto-opendata-python/

**recommendation**: use patent-client for its ORM-like interface and multi-source support (USPTO + EPO).

## anti-patterns

### scraping without permission

> "Scraping websites like Google Patents may breach their terms of service." [2]

> "Always read the terms of service—if they explicitly prohibit scraping or automated access, you're taking a legal risk." [19]

### ignoring rate limits

> "Sending thousands of requests per second can constitute a denial-of-service attack; implement proper rate limiting in your scraper with delays between requests and respect any rate limits specified in API documentation." [19]

### chatty API design

> "Chatty APIs that require the client to make too many requests to perform a single task, typically when the API provides small amounts of data in each response requiring multiple calls to gather complete information." [18]

**anti-pattern for patenter**: do not fetch patent metadata, then claims, then figures in separate calls. batch where possible.

### not versioning

> "Not versioning your API can lead to issues when you need to make breaking changes." [18]

**note**: USPTO ODP beta is shutting down April 2026 — version migration is real.

## conflict detection

### rate limits not publicly documented for ODP

sources [1], [6], [7] reference USPTO ODP but do not specify rate limits. source [8] specifies PatentsView rate limits (45/min) but this is a different API. the ODP rate limits need to be discovered via testing or contacting USPTO.

### bulk data vs API guidance varies

source [17] suggests bulk download for "large datasets" but does not define threshold. source [19] says "high-volume needs" without quantification. **recommendation**: define threshold based on actual rate limits once discovered.

## recommendations for patenter

1. **primary API**: USPTO Open Data Portal for search and document fetch
2. **secondary API**: PatentsView for structured analytics (inventor networks, citations)
3. **client library**: patent-client Python package for unified interface
4. **rate limit strategy**: exponential backoff with jitter, cache aggressively
5. **avoid**: scraping Google Patents, ignoring ToS, chatty API patterns
6. **future scope**: EPO OPS for international patent support
