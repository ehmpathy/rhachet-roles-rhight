## domain objects

### PatentSearchQuery

```typescript
interface PatentSearchQuery {
  text: string;        // 3-1000 chars, sanitized
  keywords: string[];  // extracted from text
  page?: {
    range: Partial<{
      since: { datePublished: string };  // >= (ISO date)
      until: { datePublished: string };  // <  (ISO date)
      uptil: { datePublished: string };  // <= (ISO date)
    }>;
    order: 'ASC' | 'DESC';  // sort by datePublished
    limit: number;          // max results per page
  };
}
```

**unique**: `text` (same query = same search)
**updatable**: none (queries are immutable)

**pagination-fns style**:
- `page.range.since` = `>=` (results with datePublished >= value)
- `page.range.until` = `<` (results with datePublished < value)
- `page.range.uptil` = `<=` (results with datePublished <= value)

### PatentSearchResult

interface PatentSearchResult {

**unique**: `exid`
**updatable**: none (results are immutable snapshots)

**unique**: `exid`

### PatentClaim

```typescript
interface PatentClaim {
  exid: number;        // claim number (1-indexed), unique within parent patent
  text: string;        // claim text
  dependsOn: RefByUnique<typeof PatentClaim> | null;  // null = independent claim
}
```

**unique**: `exid` (scoped to parent PatentDocument)
**updatable**: none (claims are immutable)
**note**: `dependsOn === null` means independent claim; no redundant boolean needed

**example**:
```typescript
// dependent claim
{ exid: 2, text: "The method of claim 1, wherein...", dependsOn: { exid: 1 } }

// independent claim
{ exid: 1, text: "A method for model compression...", dependsOn: null }
```

### PatentFigure

## domain relationships

| object | relates to | relationship |
|--------|------------|--------------|
| PatentSearchQuery | PatentSearchResult | 1:many (query yields results) |
| PatentSearchResult | PatentDocument | 1:1 (result.exid → document.exid) |
| PatentDocument | PatentClaim | 1:many (document has claims) |
| PatentDocument | PatentFigure | 1:many (document has figures) |
| PatentDocument | PatentMetadata | 1:1 (document has metadata) |
| PatentProposalRoute | PatentDocument | many:many (route references documents) |

## cache semantics

patents are **immutable after publication**. this enables aggressive cache:

| operation | cache strategy |
|-----------|----------------|
| search | cache by query hash, TTL 24h (results may change as new patents publish) |
| fetch | cache by exid, no TTL (patent content never changes) |

cache location: `~/.cache/rhachet/patents/{exid}.json`
