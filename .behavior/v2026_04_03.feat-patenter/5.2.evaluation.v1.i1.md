## divergences

| divergence | severity | resolution |
|------------|----------|------------|
| keyrack.yml added | backup | not in blueprint but needed for API key management. follows mechanic pattern. |
| __snapshots__/ dirs added | backup | not in blueprint but required by jest snapshot tests. standard practice. |
| API changed (USPTO ODP → PatentsView) | backup | blueprint specified USPTO ODP but research found PatentsView more suitable. documented in 3.1 research. |
| exid regex expanded (7-8 → 7-11 digits) | backup | blueprint said 7-8 digits but USPTO formats include 11-digit application numbers. more permissive is correct. |
| parse_document simplified | backup | blueprint listed prosecutionHistory extraction but PatentsView API does not provide it. documented limitation. |
| output.sh expanded | backup | blueprint listed 4 core functions, but implementation added 5 more (print_tree_error, print_alert, print_suggestion, print_section_header, print_route_info) for complete treestruct output. all follow the same pattern. |
