### patent document cache

**what**: local file cache for fetched patent documents.

**why high impact**:
- patents are immutable after publication — perfect cache candidates
- 70% of verification time is network wait
- first fetch: 5s, subsequent: <100ms
- full test suite: 60s → <5s after cache

**implementation sketch**:
```bash
CACHE_DIR="${XDG_CACHE_HOME:-$HOME/.cache}/rhachet/patents"
CACHE_FILE="$CACHE_DIR/$EXID.json"

if [[ -f "$CACHE_FILE" ]]; then
  cat "$CACHE_FILE"
  exit 0
fi

RESULT=$(curl -s "$API_URL/$EXID" ...)
mkdir -p "$CACHE_DIR"
echo "$RESULT" > "$CACHE_FILE"
echo "$RESULT"
```

**effort**: 1-2 hours

### fixture generation tool

**what**: command to capture live API responses as test fixtures.

**why medium impact**:
- manual copy-paste: 5-10 min per fixture
- capture tool: 30s per fixture
- with 10+ fixtures needed: saves 1-2 hours

**implementation sketch**:
```bash
curl -s "$SEARCH_API?q=$QUERY" | jq '.' > __fixtures__/search-$SLUG.json
curl -s "$FETCH_API/$EXID" | jq '.' > __fixtures__/patent-$EXID.json
```

**effort**: 30 min
