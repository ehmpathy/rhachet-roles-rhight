# 3.1.1.research.external.product.domain._.v1

research on the patent domain objects, operations, and relationships required for the patenter role.

---

## summary

the patent domain is hierarchical with patents as the core entity. patents contain claims (independent and dependent), specifications, figures, and abstracts. patents connect to inventors, assignees, and examiners. prosecution creates office actions, responses, and amendments. prior art forms citation networks (forward and backward). classification systems (CPC, IPC) organize patents by technology area.

key domain objects:
1. **patent** — the core entity with unique identifier (exid)
2. **claim** — legal scope definition (independent or dependent)
3. **priorArt** — referenced documents (patents, publications)
4. **officeAction** — examiner communication (non-final, final, allowance)
5. **inventor** and **assignee** — people and organizations

## domain objects

### entities (with identity and lifecycle)

#### patent

> "A patent for an invention is the grant of a property right to the inventor, issued by the United States Patent and Trademark Office." [1]

| attribute | type | description |
|-----------|------|-------------|
| exid | string | external identifier (e.g., US20210234567A1) |
| title | string | invention title (max 500 chars) |
| abstract | string | summary (max 150 words) |
| dateFiled | date | application submission date |
| datePublished | date | publication date |
| dateGranted | date | grant date (if granted) |
| status | enum | pending, granted, abandoned, expired, lapsed |

> "The front page of a granted U.S. patent includes information about its term, and a utility patent's term is generally 20 years from the application's filing date." [2]

#### patentApplication

> "A patent application is a request pending at a patent office for the grant of a patent for an invention." [3]

| attribute | type | description |
|-----------|------|-------------|
| applicationNumber | string | USPTO serial number |
| filingDate | date | date submitted |
| applicationType | enum | utility, design, plant, provisional |
| status | enum | pending, allowed, abandoned |
| technologyCenter | string | TC assignment (e.g., TC1600) |
| artUnit | string | examination unit |

> "Your application is assigned to a Technology Center and a specific patent examiner with expertise in your field. You'll get an Art Unit number." [4]

#### claim

> "The claims section of a patent application contains the types of claims which define the scope of legal protection granted by the patent." [5]

| attribute | type | description |
|-----------|------|-------------|
| number | integer | claim number (1-indexed) |
| text | string | full claim text |
| claimType | enum | independent, dependent |
| claimCategory | enum | method, system, apparatus, composition, markush |
| parentClaim | ref | for dependent claims, ref to parent |

> "Independent claims provide standalone definitions of the invention without referencing other claims. Dependent claims incorporate all limitations of an antecedent claim while adding further restrictions." [6]

#### inventor

> "Inventors are the individuals responsible for creating the matter that protection is sought for." [7]

| attribute | type | description |

#### assignee

> "Assignees are included in the bibliographic data section of patent applications. For U.S. patents, assignee types include codes such as: United States company or corporation; foreign company or corporation; United States individual; foreign individual." [7]

| attribute | type | description |
|-----------|------|-------------|
| name | string | organization or individual name |
| assigneeType | enum | usCompany, foreignCompany, usIndividual, foreignIndividual, usGovernment, foreignGovernment |

#### officeAction

> "Patent prosecution is the interaction between applicants and a patent office with regard to a patent application." [8]

| attribute | type | description |
|-----------|------|-------------|
| date | date | action date |
| actionType | enum | nonFinal, final, restriction, allowance, advisory |
| rejections | array | list of rejections with bases |

| attribute | type | description |

#### priorArtReference

> "A backward citation is a citation of a previously published document that was already available publicly before the new patent application was filed. It is often referred to as prior art." [10]

| attribute | type | description |
|-----------|------|-------------|

```
├── bibliographic

```

```
├── phase.filing
│   └── applicationFiledEvent
├── phase.examination

│   ├── response.applicant
```

## composition for wish

### prior art research flow

```
inventor.idea
  → getAllPatentsByQuery(keywords)
  → forEach(patent)
      → getOnePatentByExid(exid)
      → getAllClaimsForPatent(exid)
      → setOnePriorArtCategorization(ref, favorable|adverse)
  → priorArt.favorable[] + priorArt.adverse[]
```

### patentability analysis flow

```
priorArt.adverse[]
  → forEach(ref)
      → map claim elements to prior art disclosures
      → identify uncovered elements (novelty)
  → claim.patentable[] (elements not in prior art)
```

### office action preparation flow

```
claim.patentable[]
  → anticipate s102 rejections (single ref anticipation)
  → anticipate s103 rejections (combination of refs)
  → anticipate s101 rejections (abstract idea)
  → strategy.officeAction (prepared responses)
```

### patent proposal route composition

```
genOnePatentProposal(idea)
  → 0.idea.md (structured template)
  → 3.1.research.prior-art (search + categorize)
  → 3.2.distill.claims (analyze novelty)
  → 3.2.distill.strategy (anticipate rejections)
  → 3.3.blueprint.patent (structure application)
  → 5.1.deliver.patent.latex (formatted output)
```

## conflict detection

### claim categorization variance

sources [5] and [6] both describe independent vs dependent claims, but source [6] adds Markush claims as a separate type while source [5] treats it as a subcategory. **recommendation**: treat Markush as a claimCategory, not a claimType.

### prosecution terminology

source [8] uses "prosecution" for all examiner-applicant interaction, while source [9] uses "office action" more specifically. **convergence**: office action is a specific event type within the broader prosecution process.

---

## convergence signals

### patent structure (4+ sources agree)

sources [5], [13], [16], [17] all describe the preamble-transition-body structure for claims and the four-part patent document structure (front page, drawings, specification, claims).

### citation direction (3+ sources agree)

sources [10], [18], [19] all distinguish backward citations (prior art) from forward citations (subsequent patents).

### maintenance fee schedule (3+ sources agree)

sources [2], [12], [20] all confirm the 3.5, 7.5, 11.5 year maintenance fee windows for U.S. utility patents.

### prosecution timeline (3+ sources agree)

sources [4], [8], [11] all describe 18-36 month average prosecution timeline with 22 months to first office action.

## anti-patterns

### domain model anti-patterns

| anti-pattern | problem | sources |
|--------------|---------|---------|
| treat claims as flat list | loses independent/dependent hierarchy | [6] |
| ignore citation direction | conflates prior art with forward citations | [10] |
| mix application and granted patent | different statuses and attributes | [2], [3] |

### operation anti-patterns

| anti-pattern | problem | sources |
|--------------|---------|---------|
| search without classification | misses relevant patents in different terminology | [14] |
| fetch without prosecution history | misses office action context | [8] |
| single query strategy | misses synonym variations | [21] |

## recommendations for patenter

1. **core entities**: model Patent, Claim, Inventor, Assignee, OfficeAction, PriorArtReference as domain objects
2. **claim hierarchy**: preserve independent/dependent relationship with parentClaim reference
3. **citation direction**: distinguish backward (prior art) from forward (subsequent) citations
4. **status lifecycle**: track application through pending → allowed → granted → maintained/lapsed
5. **classification**: support both CPC and IPC for search and categorization
6. **prosecution history**: include office actions with rejection bases mapped to claims
