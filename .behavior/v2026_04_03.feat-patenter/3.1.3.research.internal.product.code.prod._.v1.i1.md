# 3.1.3.research.internal.product.code.prod._.v1

research on internal production code patterns relevant to the patenter role.

---

## summary

this research catalogs extant production code patterns in the rhachet-roles-rhight repo that the patenter role should leverage. patterns are categorized as [REUSE], [EXTEND], or [REPLACE].

key findings:
1. **role definition pattern** — Role.build() with slug, name, purpose, readme, boot, briefs, skills, inits, hooks
2. **skill structure pattern** — header doc, arg parsing, mode plan/apply, subcommand dispatch
3. **route instantiation pattern** — create directory, copy templates, bind to branch
4. **output formatting pattern** — turtle vibes with mascot header, tree structure, coconut footer
5. **domain operations pattern** — shared .operations.sh sourced by multiple skills
6. **init dispatcher pattern** — executable that configures permissions, hooks, settings

## pattern.3 = skill structure

**status**: [EXTEND]

**location**: `src/domain.roles/mechanic/skills/git.commit/git.commit.set.sh`

**pattern**:
```bash
#!/usr/bin/env bash
######################################################################
# .what = one line description
#
# .why  = why this skill exists
#         - benefit 1
#         - benefit 2
#
# usage:
#   skill.sh --arg "value"           # example invocation
#   skill.sh --mode apply            # execute mode
#
# guarantee:
#   - what this skill guarantees
#   - safety properties
######################################################################
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
source "$SCRIPT_DIR/output.sh"

MODE="plan"  # default to plan

# parse args, ignoring rhachet passthrough
while [[ $# -gt 0 ]]; do
  case $1 in
    --skill|--repo|--role) shift 2 ;;
    --mode) MODE="$2"; shift 2 ;;
    *) echo "unknown: $1"; exit 1 ;;
  esac
done

# plan mode shows preview
if [[ "$MODE" == "plan" ]]; then
  print_turtle_header "heres the wave..."
  # show what would happen
  exit 0
fi

# apply mode executes
print_turtle_header "cowabunga!"
# do the work
```

**extend for patenter**:
- same header documentation pattern
- same mode plan/apply pattern
- same rhachet arg passthrough handling
- different mascot (🦅 eagle instead of 🐢 turtle)
- different output functions for eagle vibes

## pattern.4 = subcommand dispatch

```bash
SKILL_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"

  exec)

    ;;
esac
```

## pattern.5 = route instantiation

**status**: [EXTEND]

**location**: `src/domain.roles/mechanic/skills/declapract.upgrade/init.sh`

**pattern**:
```bash
ISO_DATE=$(date +%Y_%m_%d)
ROUTE_SLUG="v${ISO_DATE}.declapract.upgrade"
ROUTE_PATH=".route/${ROUTE_SLUG}"
TEMPLATES_DIR="$SKILL_DIR/declapract.upgrade/templates"

# check if route already extant
if [[ -d "$ROUTE_PATH" ]]; then
  print_error "route already extant: $ROUTE_PATH"
  exit 1
fi

# create route directory
mkdir -p "$ROUTE_PATH"

# copy templates
for file in "$TEMPLATES_DIR"/*.stone "$TEMPLATES_DIR"/*.guard; do
  if [[ -f "$file" ]]; then
    cp -f "$file" "$ROUTE_PATH/"
  fi
done

# bind route to branch
npx rhachet run --repo bhrain --skill route.bind.set --route "$ROUTE_PATH" > /dev/null 2>&1 || true

# output success
print_turtle_header "radical!"
print_tree_start "declapract.upgrade init"
print_tree_branch "route" "$ROUTE_PATH ✨"
# ...
```

**extend for patenter**:
- same route creation pattern with different slug (v{date}.patent.propose)
- different templates (0.idea.md, research stones, distill stones, etc.)
- different mascot output (🦅 eagle)
- support --open flag to open 0.idea.md in editor

## pattern.6 = output formatting (turtle vibes)

**status**: [EXTEND]

**location**: `src/domain.roles/mechanic/skills/declapract.upgrade/output.sh`

**pattern**:
```bash
print_turtle_header() {
  echo "🐢 $1"
  echo ""
}

print_tree_start() {
  echo "🐚 $1"
}

print_tree_branch() {
  local key="$1"
  local value="$2"
  local is_last="${3:-false}"
  if [[ "$is_last" == "true" ]]; then
    echo "   └─ $key: $value"
  else
    echo "   ├─ $key: $value"
  fi
}

print_coconut() {
  echo ""
  echo "🥥 $1"
  if [[ -n "$2" ]]; then
    echo -e "   └─ \033[2m$2\033[0m"
  fi
}

print_error() {
  echo "🐢 bummer dude"
  echo ""
  echo "   └─ error: $1"
}
```

**extend for patenter**:
- replace 🐢 with 🦅 (eagle mascot for rhight repo)
- replace turtle vibes with eagle vibes:
  - "take to the sky" instead of "cowabunga"
  - "what peaks can we claim?" instead of route path message
  - "we'll track it down" instead of branch bound message
- same tree structure functions
- 🏔️ and 🌎 as secondary emojis instead of 🐚 and 🥥

## pattern.8 = init dispatcher

**status**: [REUSE]

**location**: `src/domain.roles/mechanic/inits/init.claude.sh`

**pattern**:
```bash
#!/usr/bin/env bash
set -euo pipefail
trap 'echo "❌ init.claude.sh failed at line $LINENO" >&2' ERR

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"

# backup extant settings before changes
SETTINGS_FILE="$GITROOT/.claude/settings.json"
if [[ -f "$SETTINGS_FILE" ]]; then
  ISODATETIME=$(date +%Y%m%dT%H%M%S)
  BACKUP_FILE="$GITROOT/.claude/settings.$ISODATETIME.bak.json"
  cp "$SETTINGS_FILE" "$BACKUP_FILE"
fi

# initialize permissions
"$SCRIPT_DIR/init.claude.permissions.sh"
```

**reuse for patenter**:
- same init.claude.sh dispatcher structure
- init.claude.permissions.jsonc for pre-approved commands
- simpler than mechanic (fewer hooks, fewer permissions)
