## decision rationale

**why sufficient?**

the patenter role uses the same patterns as other roles in this repo:
- bash skills with treestruct output
- jest integration tests with test-fns
- rhachet role/skill structure

all infrastructure is extant. the two identified improvements (cache, fixtures) are product implementation details, not factory prerequisites.

**why not build factory first?**

the factory upgrades distillation explicitly states: "the improvements integrate into the normal build flow — no separate factory phase needed."

a separate factory build phase would:
- delay product work unnecessarily
- create artificial phase boundaries
- violate the "no factory blockers" conclusion
