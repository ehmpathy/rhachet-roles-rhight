# playtest: patenter role

## prerequisites

1. **keyrack credentials unlocked**: `rhx keyrack unlock --owner ehmpath --env test`
2. **repo built**: `npm run build`
3. **roles linked**: `npx rhachet roles link --role patenter`
4. **network access**: USPTO API requires internet

## sandbox

all route creation targets `.temp/` within a test repo:

```bash
# create sandbox
mkdir -p .temp/playtest-patenter
cd .temp/playtest-patenter
git init
```

---

## happy paths

### 1. search for prior art

**action:**
```bash
rhx patent.priors.search --query "drill bit rock cutting"
```

**expected outcome:**
- output starts with `🦅` mascot header
- shows `patent.priors.search` section
- lists query and result count
- shows patent results with exids and titles
- exit code 0

**pass criteria:**
- `🦅` appears in first line
- at least one patent result shown (or "no results" message)
- no error messages

---

### 2. search with date filters

**action:**
```bash
rhx patent.priors.search --query "drill bit rock cutting" --since 2020-01-01 --limit 5
```

**expected outcome:**
- results limited to 5
- results from 2020 or later
- exit code 0

**pass criteria:**
- result count <= 5
- dates shown are after 2020-01-01

---

### 3. fetch a patent by exid

**action:**
```bash
# use an exid from search results (8-digit application number)
rhx patent.priors.fetch --exid 19399196
```

**expected outcome:**
- output starts with `🦅 got one,`
- shows `🌎 patent.priors.fetch` section
- shows exid, title, inventor, filed date
- cache populated at `.cache/patents/19399196/` (0.overview.meta.json, prosecution PDFs + markdown)
- exit code 0

**pass criteria:**
- `🦅` appears in first line
- title and inventor shown
- `.cache/patents/19399196/0.overview.meta.json` exists after run
- `.cache/patents/19399196/1.event.*.md` files exist (OCR transcriptions)

---

### 4. propose a new patent route

**action (in sandbox):**
```bash
cd .temp/playtest-patenter
rhx patent.propose
```

**expected outcome:**
- output starts with `🦅 take to the sky,`
- shows route directory created
- lists all 9 stone files
- shows branch bound message
- exit code 0

**pass criteria:**
- `.route/v{date}.patent.propose/` directory exists
- contains: 0.idea.md, 1.vision.stone, 3.1.research.*.stone, 3.2.distill.*.stone, 3.3.blueprint.patent.stone, 5.1.deliver.patent.latex.stone
- `.branch/.bind/{branch}` symlink exists

---

### 5. propose with --open editor

**action (in sandbox with fresh git repo):**
```bash
mkdir -p .temp/playtest-patenter-open
cd .temp/playtest-patenter-open
git init
rhx patent.propose --open cat  # use cat as harmless "editor" for test
```

**expected outcome:**
- route created successfully
- cat outputs contents of 0.idea.md
- exit code 0

**pass criteria:**
- 0.idea.md template contents displayed
- template includes structured sections for invention description

## edgey paths

### e1. search with too short query

**action:**
```bash
rhx patent.priors.search --query "ab"
```

**expected behavior:**
- error message about query too short (minimum 3 characters)
- exit code 2

**pass criteria:**
- contains "too short" or "3 character"
- does not attempt API call

---

### e2. search without query

**action:**
```bash
rhx patent.priors.search
```

**expected behavior:**
- error message about query required
- exit code 2

**pass criteria:**
- contains "query required"

---

### e3. fetch with invalid exid format

**action:**
```bash
rhx patent.priors.fetch --exid "US20210234567A1"
```

**expected behavior:**
- error message about invalid application number format
- explains that 8-digit application numbers are expected
- exit code 2

**pass criteria:**
- contains "invalid application number format"
- shows example like "19399196"

---

### e4. fetch without keyrack credentials

**action:**
```bash
# if keyrack not unlocked, skill should fail gracefully
# (to test, would need to lock keyrack or use fresh env)
rhx patent.priors.fetch --exid 19399196
```

**expected behavior (if keyrack locked):**
- error message about keyrack failed
- shows keyrack error details
- exit code 2

**pass criteria:**
- contains "keyrack failed" or "keyrack returned no secret"
- shows original keyrack error for diagnosis

---

### e5. propose when route extant

**action (in sandbox with prior route):**
```bash
cd .temp/playtest-patenter
# route already exists from happy path 4
rhx patent.propose
```

**expected behavior:**
- error message about route extant
- shows path to extant route
- exit code 2

**pass criteria:**
- contains "already" or "extant"
- shows route path

## pass/fail summary

| test | pass if | fail if |
|------|---------|---------|
| search | results or empty with 🦅 header | error, no mascot, exit != 0 |
| fetch | patent with claims and 🦅 header | error, no mascot, exit != 0 |
| propose | route created, 9 stones, symlink | route absent, files absent, exit != 0 |
| edge cases | appropriate errors, exit 2 | silent failures, exit 0 on error |
| help | usage shown, exit 0 | no output, exit != 0 |

**overall pass:** all happy paths work + all edge cases return appropriate errors
**overall fail:** any happy path fails OR edge cases silently succeed when they should error
