# playtest: patenter role

## prerequisites

- git repository (any repo with `.git/` directory)
- `PATENTSVIEW_API_KEY` environment variable set (get key at https://patentsview.org/apis/keyrequest)
- rhachet installed (`npm install` in rhachet-roles-rhight)

## sandbox

all route creation targets `.temp/` within a test repo:

```bash
# create sandbox
mkdir -p .temp/playtest-patenter
cd .temp/playtest-patenter
git init
```

---

## happy paths

### 1. search for prior art

**action:**
```bash
rhx patent.priors.search --query "neural network model compression"
```

**expected outcome:**
- output starts with `🦅` mascot header
- shows `patent.priors.search` section
- lists query and result count
- shows patent results with exids and titles
- exit code 0

**pass criteria:**
- `🦅` appears in first line
- at least one patent result shown (or "no results" message)
- no error messages

---

### 2. search with date filters

**action:**
```bash
rhx patent.priors.search --query "machine learning" --since 2020-01-01 --limit 5
```

**expected outcome:**
- results limited to 5
- results from 2020 or later
- exit code 0

**pass criteria:**
- result count <= 5
- dates shown are after 2020-01-01

---

### 3. fetch a patent by exid

**action:**
```bash
# use an exid from search results, or use known valid exid
rhx patent.priors.fetch --exid US20210234567A1
```

**expected outcome:**
- output starts with `🦅` mascot header
- shows patent title, abstract, claims
- shows metadata (filed date, inventors)
- exit code 0

**pass criteria:**
- `🦅` appears in first line
- claims section present with numbered claims
- metadata present

---

### 4. propose a new patent route

**action (in sandbox):**
```bash
cd .temp/playtest-patenter
rhx patent.propose
```

**expected outcome:**
- output starts with `🦅 take to the sky,`
- shows route directory created
- lists all 9 stone files
- shows branch bound message
- exit code 0

**pass criteria:**
- `.route/v{date}.patent.propose/` directory exists
- contains: 0.idea.md, 1.vision.stone, 3.1.research.*.stone, 3.2.distill.*.stone, 3.3.blueprint.patent.stone, 5.1.deliver.patent.latex.stone
- `.branch/.bind/{branch}` symlink exists

---

### 5. propose with --open editor

**action (in sandbox with fresh git repo):**
```bash
mkdir -p .temp/playtest-patenter-open
cd .temp/playtest-patenter-open
git init
rhx patent.propose --open cat  # use cat as harmless "editor" for test
```

**expected outcome:**
- route created successfully
- cat outputs contents of 0.idea.md
- exit code 0

**pass criteria:**
- 0.idea.md template contents displayed
- template includes structured sections for invention description

## edgey paths

### e1. search with too short query

**action:**
```bash
rhx patent.priors.search --query "ab"
```

**expected behavior:**
- error message about query too short (minimum 3 characters)
- exit code 2

**pass criteria:**
- contains "too short" or "3 character"
- does not attempt API call

---

### e2. search without query

**action:**
```bash
rhx patent.priors.search
```

**expected behavior:**
- error message about query required
- exit code 2

**pass criteria:**
- contains "query required"

---

### e3. fetch with invalid exid format

**action:**
```bash
rhx patent.priors.fetch --exid "INVALID123"
```

**expected behavior:**
- error message about invalid format
- shows expected formats
- exit code 2

**pass criteria:**
- contains "invalid" and "format"
- shows example like "US12345678A1"

---

### e4. fetch without API key

**action:**
```bash
unset PATENTSVIEW_API_KEY
rhx patent.priors.fetch --exid US20210234567A1
```

**expected behavior:**
- error message about API key required
- shows URL to get key
- exit code 2

**pass criteria:**
- contains "API key required"
- contains "patentsview.org"

---

### e5. propose when route extant

**action (in sandbox with prior route):**
```bash
cd .temp/playtest-patenter
# route already exists from happy path 4
rhx patent.propose
```

**expected behavior:**
- error message about route extant
- shows path to extant route
- exit code 2

**pass criteria:**
- contains "already" or "extant"
- shows route path

---

### e6. propose outside git repo

**action:**
```bash
mkdir -p .temp/playtest-nogit
cd .temp/playtest-nogit
rhx patent.propose
```

**expected behavior:**
- error message about git repository required
- exit code 2

**pass criteria:**
- contains "git repository"

---

### e7. --help on all skills

**action:**
```bash
rhx patent.priors.search --help
rhx patent.priors.fetch --help
rhx patent.propose --help
```

**expected behavior:**
- each shows usage info
- each exits 0

**pass criteria:**
- all three show "usage"
- all three exit 0

---

### e8. search with query too long

**action:**
```bash
rhx patent.priors.search --query "$(printf 'a%.0s' {1..1001})"
```

**expected behavior:**
- error message about query too long
- exit code 2

**pass criteria:**
- contains "too long" or "1000"
- does not attempt API call

---

### e9. fetch with valid format but nonexistent patent

**action:**
```bash
rhx patent.priors.fetch --exid US00000000A1
```

**expected behavior:**
- error message about patent not found
- exit code 2

**pass criteria:**
- contains "not found"
- shows the exid that was not found

## pass/fail summary

| test | pass if | fail if |
|------|---------|---------|
| search | results or empty with 🦅 header | error, no mascot, exit != 0 |
| fetch | patent with claims and 🦅 header | error, no mascot, exit != 0 |
| propose | route created, 9 stones, symlink | route absent, files absent, exit != 0 |
| edge cases | appropriate errors, exit 2 | silent failures, exit 0 on error |
| help | usage shown, exit 0 | no output, exit != 0 |

**overall pass:** all happy paths work + all edge cases return appropriate errors
**overall fail:** any happy path fails OR edge cases silently succeed when they should error
