# 3.3.1.blueprint.product.v1

product blueprint for the patenter role.

---

## summary

build the patenter role with three skills (`patent.priors.search`, `patent.priors.fetch`, `patent.propose`) and briefs for patent preparation best practices. the role uses the 🦅 eagle mascot and follows rhachet role conventions.

```
[+] patent.priors.search.sh
    │
    ├─ [+] parse_args()
    │      # --help: print usage and exit 0
    │      # --query "$TEXT" --limit N --since YYYY-MM-DD --until YYYY-MM-DD
    │      # extract into: QUERY_TEXT, PAGE_LIMIT (default 20), DATE_SINCE, DATE_UNTIL
    │
    ├─ [+] validate_query()
    │      # if len < 3: exit 2 "query too short"
    │      # if len > 1000: exit 2 "query too long"
    │      # sanitize: trim whitespace, escape special chars for URL
    │      # extract keywords: split on whitespace, filter stopwords
    │
    ├─ [+] search_uspto()
    │      # endpoint: GET https://data.uspto.gov/apis/patent-file-wrapper/search
    │      # headers: X-API-KEY: $USPTO_ODP_API_KEY
    │      # params: q=$QUERY_TEXT (OpenSearch syntax)
    │      #         rows=$PAGE_LIMIT
    │      #         start=0
    │      # if DATE_SINCE: add datePublished:[$DATE_SINCE TO *]
    │      # if DATE_UNTIL: add datePublished:[* TO $DATE_UNTIL]
    │      # curl with 10s timeout
    │      # if 429: read Retry-After header, sleep, retry with exponential backoff + jitter
    │      # if network error: exit 1 "API unavailable: check network connection"
    │
    ├─ [+] parse_results()
    │      # jq: .results[] | {exid:.applicationNumber, title:.title,
    │      #     abstract:.abstract, datePublished:.publicationDate}
    │      # if .results null or empty:
    │      #   set RESULTS=[] and ALERT="no results found"
    │      #   set SUGGESTION="try broader terms, synonyms, or check for typos"
    │      # if query has <3 keywords: set ALERT="query may be too vague"
    │
    ├─ [+] sort_by_relevance()
    │      # compute relevance: count keyword matches in title (2x) + abstract (1x)
    │      # scale to 0-1: score / max_possible
    │      # sort descending by relevance
    │
    ├─ [+] emit_results()
    │      # 🦅 lets soar and see,

    │      #    ├─ query: $QUERY_TEXT

    │      #       ├─ US20210234567A1 (0.87)
    │      #       │  └─ Neural network compression via pruning
    │      #       └─ ...
    │      # if ALERT: print after results
    │      # if SUGGESTION: print after ALERT
    │
    └─ [←] output.sh
```

### patent.priors.fetch

```
[+] patent.priors.fetch.sh
    │
    ├─ [+] parse_args()
    │      # --help: print usage and exit 0
    │      # --exid US20210234567A1
    │      # extract into: PATENT_EXID
    │
    ├─ [+] validate_exid()
    │      # regex: ^US[0-9]{7,8}(A1|B1|B2)?$
    │      # if no match: exit 2 "invalid patent format, expected US12345678A1"
    │
    ├─ [+] cache_get()
    │      # CACHE_PATH=~/.cache/rhachet/patents/${PATENT_EXID}.json
    │      # if file extant: cat and return, skip fetch
    │      # patents immutable → cache never stale
    │
    ├─ [+] fetch_patent()
    │      # endpoint: GET https://data.uspto.gov/apis/patent-file-wrapper/documents/$PATENT_EXID
    │      # headers: X-API-KEY: $USPTO_ODP_API_KEY
    │      # curl with 10s timeout
    │      # if 429: read Retry-After header, sleep, retry with exponential backoff + jitter
    │      # if 404: exit 2 "patent not found: $PATENT_EXID. verify exid format (e.g., US12345678A1)"
    │      # if network error: exit 1 "API unavailable: check network connection"
    │
    ├─ [+] parse_document()

    │      #   title: .title
    │      #   abstract: .abstract

    │      #   metadata: {dateFiled, datePublished, inventors, assignee, classifications}
    │      # figures: .drawings[] if available
    │      # history: .prosecutionHistory[] if available
    │
    ├─ [+] cache_set()
    │      # mkdir -p ~/.cache/rhachet/patents/
    │      # write JSON to $CACHE_PATH
    │
    ├─ [+] emit_document()

    │      #    ├─ exid: US20210234567A1

    │      #    ├─ claims: 12 (3 independent)
    │      #    │  ├─ 1. A method for model compression... [independent]

    │      #    └─ classifications: G06N3/08, H04L9/00
```

## test coverage

### mandatory requirements

1. **`--help` on every skill** — each skill must support `--help` flag that displays usage info
2. **stdout snapshots on every test** — every `then()` block must include `expect(result.stdout).toMatchSnapshot()` for vibecheck in PR reviews

### integration tests

| skill | test file | coverage |
|-------|-----------|----------|
| patent.priors.search | `patent.priors.search.integration.test.ts` | help, query too short, no query, valid query |
| patent.priors.fetch | `patent.priors.fetch.integration.test.ts` | help, no exid, invalid format, valid exid |
| patent.propose | `patent.propose.integration.test.ts` | help, route creation, branch bind, extant route, invalid editor |

### snapshot coverage

| journey | snapshots |
|---------|-----------|
| search t1 success | results tree with relevance scores |
| search t2 vague query | results + alert |
| search t3 no results | empty + suggestions |
| fetch t1 success | document tree with claims |
| fetch t2 invalid exid | error message |
| fetch t3 not found | error + verification hint |
| propose t1 success | route tree with all stones |
| propose t2 already extant | error message |

### unit tests

no unit tests needed — skills are thin CLI wrappers around curl + jq. logic is minimal. integration tests cover all paths.

## build order

1. **briefs first** — establish terminology and rules before code
2. **output.sh** — shared output functions with 🦅 mascot
3. **patent.priors.fetch** — simpler than search, validate API access
4. **patent.priors.search** — uses same API patterns as fetch
5. **patent.propose** — route creation, depends on templates
6. **integration tests** — verify all journeys
7. **role definition** — getPatenterRole.ts, boot.yml, readme.md

---

## exit criteria

- [ ] all three skills implemented and tested
- [ ] briefs provide enough context for patent preparation
- [ ] integration tests pass with fixtures
- [ ] role boots successfully in claude session
- [ ] 🦅 mascot output is consistent and vibey
