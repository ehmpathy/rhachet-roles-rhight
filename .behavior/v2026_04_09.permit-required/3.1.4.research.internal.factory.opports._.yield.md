# research: factory opportunities

## summary

the singular bottleneck is **scrape infrastructure** — we have no reusable browser automation setup in this repo. one build of this module would unblock permit skills and benefit future legal research roles.

---

## the constraint

### THE bottleneck: browser automation setup

the controllable constraint is **absence of reusable scrape infrastructure**.

| constraint | controllable | impact |
|------------|--------------|--------|
| Accela API access | no | blocked by Indianapolis DBNS |
| portal scrape setup | yes | upfront investment, then reusable |
| fixture creation | yes | one-time per test case |

**if we fix one problem**: build a Playwright-based scrape module that permit.search and permit.fetch can use. this unblocks:
- permit.search (address lookup)
- permit.fetch (permit detail retrieval)
- future jurisdiction expansions

### queues in verification

| queue | wait time | controllable |

can run 3-5 parallel scrape requests before portal may rate limit.

## waste

### handoffs that slow down

| handoff | waste | fix |
|---------|-------|-----|
| scrape → parse | format mismatch | standardize response shape |
| parse → cache | manual file management | automate cache write |
| cache → test | fixture sync | use genTempDir clone pattern |

### repeated setup

| setup | frequency | cache opportunity |
|-------|-----------|------------------|
| browser launch | every scrape | reuse browser context |
| portal navigation | every search | keep session warm |
| selector definition | every UI change | version selectors |

### idle partial work

| partial work | resolution |
|--------------|------------|
| scraped but not cached | write-through cache |
| cached but not tested | fixture generation |
| tested but not integrated | skill composition |

### manual error-prone tasks

| task | error mode | automation |
|------|------------|-----------|
| selector updates | silent breakage | CI selector validation |
| cache invalidation | stale data | TTL-based expiry |
| fixture creation | absent cases | auto-generate from live scrape |

## recommended opportunity

**build a reusable Playwright-based scrape module**:

```
src/domain.roles/permiter/skills/permit.search/
├── scrape/
│   ├── launchBrowser.ts       # reusable browser context
│   ├── navigateToSearch.ts    # portal navigation
│   ├── enterSearchCriteria.ts # form fill
│   ├── extractResults.ts      # result parse
│   └── selectors.ts           # version-controlled selectors
```

this module can be:
1. tested independently (unit tests for each function)
2. cached at each stage (HTML, parsed data, final result)
3. reused by permit.fetch and future skills
4. updated when selectors change
