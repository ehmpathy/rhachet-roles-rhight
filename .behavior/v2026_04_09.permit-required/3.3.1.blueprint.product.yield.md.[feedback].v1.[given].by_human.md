            # howto: create thought routes

            ## .what
            guide for create new thought routes: directories, stones, guards, and bind commands.

            ## .why
            enable self-serve route creation without reverse-engineer extant routes.

            ---

            ## concepts

            | concept | what | file |
            |---------|------|------|
            | route | sequence of milestones | `.route/v$isodate.$slug/` |
            | stone | single milestone | `N.name.stone` |
            | guard | validation gate (`artifacts:`, `protect:`, `judges:`, `reviews:`) | `N.name.guard` |

            ---

            ## workflow

            1. **create** route directory: `.route/v$isodate.$slug/`
            2. **add** stones: `1.name.stone`, `2.name.stone`, etc.
            3. **add** guards (optional): `1.name.guard`, etc.
            4. **bind** route to branch: `rhx route.bind.set --route .route/v$isodate.$slug`

            **critical**: step 4 is required. without a bind, the driver won't know the route exists.

            ### skills that initialize routes

            skills that create routes MUST also bind them:

            ```sh
            rhx route.bind.set --route "$ROUTE_PATH"
            ```

            if a skill creates a route but does not bind it, the route will be orphaned.

            ## commands

            | command | purpose |
            |---------|---------|
            | `rhx route.bind.set --route $path` | bind route |
            | `rhx route.bind.get` | show bound route |
            | `rhx route.bind.del` | unbind |
            | `rhx route.drive` | show next stone |
            | `rhx route.stone.set --stone $name --as passed` | mark complete |

            ---

            ## guard syntax

            | section | what |
            |---------|------|
            | `artifacts:` | files stone must produce |
            | `protect:` | files stone must not modify until passed |
            | `judges:` | commands to verify driver can pass gate |
            | `reviews:` | peer + self reviews |

            ```yaml
            artifacts:
              - $route/3.1.a*.md

            protect:
              - src/**/*

            judges:
              - rhx route.stone.judge --mechanism reviewed? --stone $stone --route $route
              - rhx route.stone.judge --mechanism approved? --stone $stone --route $route

            reviews:
              peer:
                - npx rhachet run --repo bhrain --skill review --output '$route/.reviews/$stone.peer-review.md'

              self:
                - slug: has-questioned-requirements
                  say: |
                    are there any requirements that should be questioned?
            ```

            ### peer vs self

            | type | format | who |
            |------|--------|-----|
            | peer | shell commands | external brains |
            | self | structured prompts (`say: \|` enables multiline) | internal brains |

            ### judges

            | mechanism | what | required |
            |-----------|------|----------|
            | `reviewed?` | peer reviewers satisfied | when peer reviews exist |
            | `approved?` | human approval received | when human sign-off needed |

            ### variables

            | var | expands to |
            |-----|------------|
            | `$route` | route directory path |
            | `$stone` | stone name |
            | `$hash` | content hash |

            ---

            ## parallel stones

            same numeric prefix = parallel: `3.1.a.stone`, `3.1.b.stone`
            route advances when ALL parallel stones pass

            ---

            ## variants

            - `.route/` — default
            - `.behavior/` — behavior-driven
            - `.$custom/` — custom

            ---

            ## nested routes

            when a stone's work requires its own route:
            1. create sub-route in separate directory
            2. rebind: `rhx route.bind.set --route .route/v$date.$subroute`
            3. complete sub-route
            4. rebind to parent: `rhx route.bind.set --route .route/v$date.$parent`
