# blueprint: product

## summary

**three skills to build**:
1. `permit.check.required` — thought route to determine if work requires permit
2. `permit.search` — shell skill to search permits by address
3. `permit.fetch` — shell skill to fetch permit details by ID

**domain layer**: 3 entities (Permit, PermitJurisdiction, PermitCodeSection), 3 literals (PermitDetermination, PermitWorkDescription, PermitCodeCitation), 9 operations (get/set patterns).

**factory dependency**: scrape module must be built first (see 3.3.0.blueprint.factory.yield.md).

## build sequence

1. **domain objects** — Permit, PermitJurisdiction, PermitCodeSection, literals
2. **transformers** — parsePermitWorkDescription, computePermitDetermination
3. **operations** — get/set for permit, permitJurisdiction, permitCode
4. **permit.search skill** — shell + scrape integration
5. **permit.fetch skill** — shell + cache + scrape
6. **permit.check.required skill** — thought route + templates
7. **integration tests** — all operations
8. **acceptance tests** — full journey snapshots
