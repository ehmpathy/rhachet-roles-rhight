# distill: factory upgrades

## decision

**one build required**: Playwright-based scrape module for permit portal access.

the extant factory is **mostly sufficient** — test infra, cache patterns, and fixture templates are reusable from patenter. the singular bottleneck is browser automation.

---

## blockers to address

| blocker | what to build | priority |
|---------|---------------|----------|
| no browser automation in repo | `src/utils/scrape/` module with Playwright | high |

### detail: scrape module

```
src/utils/scrape/
├── launchBrowser.ts       # browser context management
├── navigateToPortal.ts    # portal-specific navigation
├── fillForm.ts            # generic form fill
├── extractTable.ts        # generic table parse
└── selectors/
    └── indianapolis.accela.ts  # versioned selectors
```

**why priority high**: without this, permit.search and permit.fetch cannot be built. this is the single bottleneck identified in `3.1.4.research.internal.factory.opports._.yield.md`.

---

## improvements to make

| opportunity | what to build | benefit |
|-------------|---------------|---------|
| selector version control | `selectors/{portal}.{version}.ts` pattern | graceful UI change handle |
| cache write-through | auto-cache on scrape success | reduce repeat requests |
| rate limit queue | p-queue integration | avoid portal blocks |

### detail: selector version control

```typescript
// selectors/indianapolis.accela.v2026-04.ts
export const SELECTORS = {
  searchForm: {
    addressInput: '#ctl00_PlaceHolderMain_addressSearch_txtStreetName',
    submitButton: '#ctl00_PlaceHolderMain_btnSearch',
  },
  results: {
    table: '.ACA_Grid_Row',
    permitNumber: 'td:nth-child(1)',
    permitType: 'td:nth-child(2)',
    status: 'td:nth-child(3)',
    date: 'td:nth-child(4)',
  },
} as const;
```

selectors are versioned by date. when UI changes, create new version file. old versions remain for reference.

## dependency: Playwright

Playwright must be added to devDependencies:

```bash
rhx set.package.install --package playwright --at 1.52.0 --for dev --reason "browser automation for permit portal scrape"
```

**note**: this is a devDependency because:
- only needed for integration tests
- only needed for live scrape (dev/test, not prod)
- prod uses cached results

---

## build sequence

1. install Playwright devDependency
2. create `src/utils/scrape/` module
3. create selector files for Indianapolis Accela
4. build permit.search skill that uses scrape module
5. build permit.fetch skill that uses scrape module
6. add integration tests with fixtures

this sequence unblocks the product build in phase 5.
