# howto.indianapolis-permit-api-search

## .what

handoff doc for programmatic permit searches in Indianapolis-Marion County.

## .why

- manual portal searches don't scale
- API access enables automated monitoring
- integration with property management workflows

---

## tl;dr

Indianapolis uses **Accela** for permits. API access requires **agency registration** (not public). alternatives:

1. **Accela API** — requires Indianapolis DBNS partnership (blocked for most)
2. **Open Indy Data Portal** — limited permit data exposed
3. **Web scraping** — works but fragile
4. **Manual portal** — reliable fallback

---

## option 1: Accela API (official)

### what it is

Accela is the SaaS platform Indianapolis uses for permits. they expose a REST API.

### endpoints

| endpoint | method | purpose |
|----------|--------|---------|
| `/v4/records` | GET | search permits |
| `/v4/records/{id}` | GET | get permit details |
| `/v4/search/records` | POST | advanced search |
| `/v4/addresses` | GET | search by address |

### authentication

```
POST https://auth.accela.com/oauth2/token
Content-Type: application/x-www-form-urlencoded

grant_type=password
&client_id={app_id}
&client_secret={app_secret}
&agency_name=INDY
&environment=PROD
&username={username}
&password={password}
&scope=records
```

### the blocker

**you need agency registration**. this means:

1. register app at [developer.accela.com](https://developer.accela.com)
2. request access to `INDY` agency environment
3. Indianapolis DBNS must approve your app
4. receive `client_id` and `client_secret`

**status**: most requests are denied unless you're a government contractor or civic app partner.

### if you get access

```bash
# search permits by address
curl -X POST "https://apis.accela.com/v4/search/records" \
  -H "Authorization: Bearer {access_token}" \
  -H "x-accela-agency: INDY" \
  -H "Content-Type: application/json" \
  -d '{
    "address": {
      "streetName": "Main",
      "streetNumber": "123"
    },
    "type": {
      "module": "Building"
    }
  }'
```

### response shape

```json
{
  "result": [
    {
      "id": "INDY-2019-00012345",
      "type": { "module": "Building", "type": "Electrical" },
      "status": { "value": "Issued" },
      "openedDate": "2019-03-15",
      "address": { "streetNumber": "123", "streetName": "Main St" },
      "contacts": [{ "type": "Contractor", "name": "ABC Electric" }]
    }
  ]
}
```

### docs

- [Accela API index](https://developer.accela.com/docs/api_reference/api-index.html)
- [Authentication](https://developer.accela.com/docs/construct-apiAuth.html)
- [Records API](https://developer.accela.com/docs/api_reference/api-records.html)
- [Search API](https://developer.accela.com/docs/api_reference/api-search.html)

## option 2: Open Indy Data Portal

### what it is

Indianapolis publishes some civic data at [data.indy.gov](https://data.indy.gov).

### relevant categories

| category | contents |
|----------|----------|
| Property | land survey, preservation info |
| Planning & Zoning | zone boundaries, land use |

### the limitation

**permit records are NOT currently exposed** in the open data portal. the portal has 123 datasets but permits are behind the Accela paywall.

### api format (for datasets that exist)

```bash
# example: if permits were exposed
curl "https://data.indy.gov/api/v2/catalog/datasets/permits/records?limit=10"
```

### check for updates

datasets change. search for "permit" or "electrical" periodically:

```
https://data.indy.gov/search?q=permit
```

## option 3: web scraping (unofficial)

### target

```
https://aca-prod.accela.com/INDY/Cap/CapHome.aspx?module=Permits
```

### approach

1. use headless browser (puppeteer, playwright)
2. navigate to search page
3. enter address
4. parse results table

### example (playwright)

```typescript
import { chromium } from 'playwright';

const searchPermits = async (address: string) => {
  const browser = await chromium.launch();
  const page = await browser.newPage();

  // navigate to permit search
  await page.goto('https://aca-prod.accela.com/INDY/Cap/CapHome.aspx?module=Permits');

  // select address search
  await page.selectOption('#ctl00_PlaceHolderMain_TabDataList_ctl01_btnGroup', 'Address');

  // enter address
  await page.fill('#ctl00_PlaceHolderMain_addressSearch_txtStreetName', address);

  // submit search
  await page.click('#ctl00_PlaceHolderMain_btnSearch');

  // wait for results
  await page.waitForSelector('.ACA_Grid_Row');

  // extract results
  const results = await page.$$eval('.ACA_Grid_Row', rows =>
    rows.map(row => ({
      permitNumber: row.querySelector('td:nth-child(1)')?.textContent?.trim(),
      type: row.querySelector('td:nth-child(2)')?.textContent?.trim(),
      status: row.querySelector('td:nth-child(3)')?.textContent?.trim(),
      date: row.querySelector('td:nth-child(4)')?.textContent?.trim(),
    }))
  );

  await browser.close();
  return results;
};
```

### caveats

- **fragile**: selectors change when Accela updates UI
- **rate limits**: too many requests may trigger blocks
- **ToS**: check if scrape violates terms
- **maintenance**: expect breakage every few months

## option 4: manual portal (fallback)

### url

```
https://aca-prod.accela.com/INDY/Cap/CapHome.aspx?module=Permits
```

### steps

1. go to portal
2. select "Address" from search dropdown
3. enter street name (e.g., "Main")
4. enter street number (e.g., "123")
5. click search
6. review results

### export

no bulk export. screenshot or copy/paste results.

| use case | approach |
|----------|----------|
| one-time lookup | manual portal |
| occasional checks | manual portal |
| bulk property checks | web scrape (with caution) |
| production integration | apply for Accela API access |
| civic app project | contact DBNS for API partnership |
