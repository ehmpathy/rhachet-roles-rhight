# blackbox criteria: permit-required

## usecase.1 = permit.check.required

### episode: determine if permit required

```
given(homeowner wants to know if permit required for work)
  when(invoke permit.check.required with postal and work description)
    then(thought route is instantiated)
      sothat(structured research can begin)
    then(route is bound to current branch)
      sothat(progress is tracked via hooks)
    then(0.query stone is pre-filled with postal and work)
      sothat(user can review and refine before research)

given(thought route is instantiated)
  when(research stones are completed)
    then(work description is mapped to code terminology)
      sothat(user sees what code terms apply to their work)
    then(baseline code is identified with verbatim text)
      sothat(user sees exact language that requires permits)
    then(exemptions are checked with verbatim text)
      sothat(user sees if their work qualifies for exemption)
    then(jurisdiction adoption is documented)
      sothat(user knows which code version applies locally)

given(research is complete)
  when(deliver stones are completed)
    then(diagnosis markdown is emitted with full analysis)
      sothat(user has thorough documentation with all citations)
    then(coverpage markdown is emitted with treestruct summary)
      sothat(user has one-page overview of determination)
    then(determination is one of: required | not_required | conditional | unclear)
      sothat(user knows the outcome clearly)
    then(disclaimer is included)
      sothat(user knows this is not legal advice)
```

### exchange: determination outcomes

```
given(work clearly requires permit per baseline code)
  when(no exemption applies)
    then(determination = required)
    then(proof chain shows: baseline match + no exemption + jurisdiction confirms)

given(work matches an exemption in R105.2)
  when(exemption is unconditional)
    then(determination = not_required)
    then(proof chain shows: exemption text verbatim)

given(work matches an exemption with conditions)
  when(condition cannot be verified from input)
    then(determination = conditional)
    then(condition field explains what determines applicability)

given(work description is ambiguous)
  when(multiple code sections could apply)
    then(determination = unclear)
    then(diagnosis lists all possible interpretations)
    then(suggested questions are provided for permit office)

given(jurisdiction code is not accessible)
  when(state or local code behind paywall)
    then(determination = unclear)
    then(diagnosis notes which sources were inaccessible)
    then(source is cited as "not publicly available")
```

## usecase.2 = permit.search

### episode: search permit history

```
given(user wants to see permits for a property)
  when(invoke permit.search with address and postal)
    then(permits are returned for that address)
      sothat(user sees what work has been permitted)
    then(each permit shows number, type, status, date, contractor)
      sothat(user can identify specific permits of interest)

given(user wants to paginate results)
  when(invoke with --limit)
    then(at most limit permits are returned)
    then(next cursor is provided if more exist)
      sothat(user can fetch additional pages)
  when(invoke with --since)
    then(only permits issued after date are returned)
  when(invoke with --until)
    then(only permits issued before date are returned)
  when(invoke with cursor from previous response)
    then(next page of results is returned)
```

### exchange: search results

```
given(address has permits on file)
  when(search is invoked)
    then(permits array is populated)
    then(source jurisdiction and portal are shown)
      sothat(user knows where data came from)

given(address has no permits on file)
  when(search is invoked)
    then(permits array is empty)
    then(source is still shown)
      sothat(user knows search succeeded but found no records)

given(address is not found in jurisdiction)
  when(search is invoked)
    then(error indicates address not found)
    then(suggestion to verify address format is provided)

given(jurisdiction portal is unavailable)
  when(search is invoked)
    then(error indicates portal unavailable)
    then(retry hint is provided)
```

## usecase.3 = permit.fetch

### episode: fetch permit details

```
given(user wants full details of a specific permit)
  when(invoke permit.fetch with permit id)
    then(permit details are returned)
      sothat(user sees complete record)
    then(inspection history is included)
      sothat(user sees progress through approval process)
```

### exchange: fetch results

```
given(permit id exists)
  when(fetch is invoked)
    then(permit number, type, status are returned)
    then(address is returned)
    then(work description is returned)
    then(contractor name is returned if on file)
    then(inspection timeline is returned)
      sothat(user sees rough-in, final, etc)

given(permit id does not exist)
  when(fetch is invoked)
    then(error indicates permit not found)
    then(suggestion to verify id format is provided)

given(permit has no inspections yet)
  when(fetch is invoked)
    then(inspections array is empty)
    then(status reflects permit stage (e.g., Issued, not yet inspected))
```

## cross-cutting: disclaimers

```
given(any output is produced)
  when(output contains legal-adjacent information)
    then(disclaimer is included)
      sothat(user knows this is not legal advice)
    then(suggestion to consult attorney or permit office is included)
      sothat(user knows next steps for authoritative guidance)
```
