# distill: domain objects and operations

## summary

the permit system requires **3 domain entities** (Permit, Jurisdiction, CodeSection), **3 domain literals** (PermitDetermination, WorkDescription, CodeCitation), and **6 domain operations** (get/set patterns for search, fetch, and determination).

---

## usecases

| usecase | actor | goal |
|---------|-------|------|
| UC-1 | homeowner | determine if work requires permit |
| UC-2 | homeowner | search permits by address |
| UC-3 | homeowner | view permit details |
| UC-4 | property buyer | verify permit history for address |
| UC-5 | contractor | check status of pulled permit |

---

## contract envisions

### UC-1: permit.check.required

```
input:
  postal: string         # ZIP code for jurisdiction lookup
  work: string           # work description in homeowner language

output:
  determination: 'required' | 'not-required' | 'unclear'
  proof:
    baseline: CodeCitation[]    # what the code says
    exemptions: CodeCitation[]  # exemptions checked
    jurisdiction: string        # which code version applied
  nextSteps:
    contact?: { name, phone, portal }  # if unclear or required
    documentation?: string             # if not required
```

### UC-2: permit.search

```
input:
  address: string        # street address
  postal?: string        # optional ZIP filter
  since?: string         # optional date filter
  limit?: number         # result limit

output:
  permits: PermitSummary[]
  total: number
```

### UC-3: permit.fetch

```
input:
  permitId: string       # permit number

output:
  permit: Permit
  documents: PermitDocument[]
```

## domain objects

### entities (identity via unique key)

#### Permit

```typescript
interface Permit {
  // unique
  permitNumber: string;           // e.g., "ELEC-2024-00012345"
  jurisdiction: Ref<Jurisdiction>;

  // data
  type: PermitType;               // 'electrical' | 'plumb' | 'mechanical' | 'build'
  status: PermitStatus;           // 'applied' | 'issued' | 'expired' | 'revoked'
  address: string;
  applicant: string | null;
  contractor: string | null;
  description: string | null;
  appliedAt: string;              // ISO date
  issuedAt: string | null;
  expiresAt: string | null;

  // metadata (database-generated)
  id?: string;
  createdAt?: string;
  updatedAt?: string;
}

class Permit extends DomainEntity<Permit> {
  public static unique = ['permitNumber', 'jurisdiction'] as const;
}
```

#### Jurisdiction

```typescript
interface Jurisdiction {
  // unique
  slug: string;                   // e.g., "indianapolis-marion-county-in"

  // data
  name: string;                   // "Indianapolis-Marion County, Indiana"
  state: string;                  // "IN"
  codeAdopted: string;            // "IRC 2020"
  codeAdoptedAt: string;          // ISO date
  portalUrl: string | null;
  contactPhone: string | null;
  contactEmail: string | null;

  // metadata
  id?: string;
  createdAt?: string;
  updatedAt?: string;
}

class Jurisdiction extends DomainEntity<Jurisdiction> {
  public static unique = ['slug'] as const;
}
```

#### CodeSection

```typescript
interface CodeSection {
  // unique
  code: string;                   // "IRC 2020"
  section: string;                // "R105.1"

  // data
  title: string;                  // "Required"
  verbatim: string;               // exact text from code
  effectiveAt: string;            // when this version took effect
  supersededAt: string | null;    // when replaced by newer version

  // metadata
  id?: string;
  createdAt?: string;
  updatedAt?: string;
}

class CodeSection extends DomainEntity<CodeSection> {
  public static unique = ['code', 'section'] as const;
}
```

### literals (identity via all properties)

#### PermitDetermination

```typescript
interface PermitDetermination {
  determination: 'required' | 'not-required' | 'unclear';
  confidence: number;             // 0-1
  baseline: CodeCitation[];
  exemptionsChecked: CodeCitation[];
  jurisdictionUsed: Ref<Jurisdiction>;
  determinedAt: string;           // ISO timestamp
}

class PermitDetermination extends DomainLiteral<PermitDetermination> {}
```

#### WorkDescription

```typescript
interface WorkDescription {
  raw: string;                    // homeowner's words
  normalized: string;             // parsed to code category
  category: 'electrical' | 'plumb' | 'mechanical' | 'build' | 'unknown';
  subcategory: string | null;     // e.g., "service-upgrade", "panel-replace"
}

class WorkDescription extends DomainLiteral<WorkDescription> {}
```

#### CodeCitation

```typescript
interface CodeCitation {
  codeRef: Ref<CodeSection>;
  verbatim: string;               // quoted text
  applies: boolean;               // does this citation apply to the work?
  reason: string;                 // why it applies or doesn't
}

class CodeCitation extends DomainLiteral<CodeCitation> {}
```

## relationships

```
WorkDescription
    └─→ category maps to CodeSection queries

Jurisdiction
    └─→ determines which CodeSection versions apply
    └─→ linked to Permit records

PermitDetermination
    ├─→ references Jurisdiction
    ├─→ contains CodeCitation[]
    │       └─→ each references CodeSection
    └─→ computed from WorkDescription + Jurisdiction + CodeSection[]

Permit
    └─→ belongs to Jurisdiction
```

### treestruct of common subdomains

```
permit-required/
├── determination/
│   ├── WorkDescription
│   ├── PermitDetermination
│   ├── CodeCitation
│   └── getOnePermitDetermination
├── code/
│   ├── CodeSection
│   └── getOneCodeSection, getAllCodeSectionsForCategory
├── jurisdiction/
│   ├── Jurisdiction
│   └── getOneJurisdiction
└── permit/
    ├── Permit
    └── getOnePermit, getAllPermitsForAddress
```

## composition for wish fulfillment

### flow: permit.check.required

```
1. parseWorkDescription(raw) → WorkDescription
2. getOneJurisdiction({ postal }) → Jurisdiction
3. getAllCodeSectionsForCategory({ category, code }) → CodeSection[]
4. computeDetermination({ work, jurisdiction, sections }) → PermitDetermination
5. formatOutput({ determination }) → treestruct output
```

### flow: permit.search

```
1. scrapePermitPortal({ address }) → raw HTML
2. parsePermitResults(html) → Permit[]
3. setPermit(permit) for each → cache
4. formatOutput({ permits }) → treestruct output
```

### flow: permit.fetch

```
1. getOnePermit({ permitNumber }) → Permit | null (from cache)
2. if not cached: scrapePermitDetail({ permitNumber }) → raw HTML
3. parsePermitDetail(html) → Permit
4. setPermit(permit) → cache
5. formatOutput({ permit }) → treestruct output
```
