# research: product premortem

## summary

the nightmare scenario is **wrong determination** — a homeowner trusts the tool, skips the permit, and faces fines, stop-work orders, or insurance voidance. mitigation: "unclear" must be a first-class outcome, not a failure mode.

---

## the nightmare scenario

### what happens

1. homeowner asks: "do i need a permit for this electrical panel upgrade?"
2. tool returns: **permit not required**
3. homeowner proceeds without permit
4. inspector discovers unpermitted work
5. homeowner faces:
   - stop-work order
   - fines ($500-$5000+)
   - insurance claim denial
   - required permit + inspection (retroactive)
   - potential property sale complications

### why it happens

| cause | mechanism |
|-------|-----------|
| wrong code interpretation | tool misreads exemption criteria |
| stale code citation | code changed since tool's knowledge cutoff |
| jurisdiction mismatch | tool applies wrong jurisdiction's rules |
| work scope misclassification | "electrical panel" parsed as "minor electrical" |

### who gets hurt

| party | harm |
|-------|------|
| homeowner | fines, delays, insurance gaps |
| tool creator | liability exposure, reputation damage |
| permit office | cleanup of unpermitted work |

## failure scenarios

### scenario 1: wrong determination

**trigger**: tool determines "permit not required" when permit IS required

**sequence**:
1. homeowner describes work as "electrical panel upgrade"
2. tool checks IRC R105.2 exemptions
3. tool incorrectly matches to an exemption
4. returns "permit not required"
5. homeowner proceeds, inspector finds unpermitted work

**root cause options**:
- exemption logic flawed
- work description ambiguous
- code citation incomplete

**mitigation**:
- "unclear" as first-class outcome
- require explicit work scope confirmation
- cite verbatim code, not paraphrases
- confidence score with threshold

### scenario 2: stale code citation

**trigger**: tool cites code version that has been superseded

**sequence**:
1. tool cites IRC 2018 R105.1
2. Indianapolis adopted IRC 2021 with amendments
3. amendment changed exemption criteria
4. tool's determination based on outdated rules

**root cause options**:
- no code version check
- no jurisdiction adoption date tracking
- no TTL on cached citations

**mitigation**:
- track code adoption dates per jurisdiction
- TTL alert if citation > 2 years old
- display "as of [date]" on all determinations
- link to official jurisdiction code source

### scenario 3: portal scrape breakage

**trigger**: Accela portal UI changes, scrape returns wrong data

**sequence**:
1. permit.search scrapes portal for permit history
2. portal updates UI, selectors break
3. scrape returns empty or partial results
4. tool reports "no permits found" when permits exist
5. user makes decisions based on false negative

**root cause options**:
- selectors not version-controlled
- no validation of scrape results
- no fallback to manual lookup

**mitigation**:
- version selectors, track last-known-good
- validate scraped data shape
- CI check for selector breakage
- degrade gracefully with "unable to verify" status

## key mitigation: "unclear" as first-class outcome

### the insight

legal research tools fail when they force binary answers. the correct frame for legal research tools:

| outcome | meaning | user action |
|---------|---------|-------------|
| **required** | high confidence permit needed | apply for permit |
| **not required** | high confidence exempt | proceed, document exemption |
| **unclear** | insufficient confidence | consult permit office |

### why "unclear" works

1. **honest**: admits tool limitations
2. **safe**: defers to authority when uncertain
3. **useful**: still narrows scope of what to ask
4. **defensible**: tool never claims false certainty

### implementation

```
determination: unclear

reason: work description matches multiple categories
  - "electrical panel upgrade" could be:
    - service upgrade (permit required)
    - panel replacement (permit required)
    - subpanel addition (permit required)
  - none of IRC R105.2 exemptions apply to panel work

recommendation: contact Indianapolis DBNS at (317) 327-8700
  - ask: "does a 100A to 200A panel upgrade require a permit?"
  - reference: IRC R105.1, electrical systems
```

## premortem checklist

before MVP launch, verify:

- [ ] "unclear" outcome implemented and tested
- [ ] code citations include version and date
- [ ] jurisdiction adoption dates tracked
- [ ] scrape selectors version-controlled
- [ ] scrape validation catches empty results
- [ ] user sees "not legal advice" disclaimer
- [ ] permit office contact info always shown
