# research: product root cause

## summary

the root cause is **legal code is inaccessible to non-experts**. homeowners cannot easily find, read, or interpret the codes that govern their work. the permit portal tells them IF a permit was pulled, not WHETHER one is required.

---

## what is the symptom?

### the observable problem

homeowners do not know if they need a permit before they start work.

### how it manifests

| manifestation | example |
|---------------|---------|
| uncertainty | "i heard i might need a permit but i'm not sure" |
| avoidance | "i'll just do it and hope nobody notices" |
| wrong action | "i asked my neighbor and they said no permit needed" |
| after-the-fact discovery | "the inspector said i should have pulled a permit" |

### when it occurs

- before a home improvement project
- when contractor quotes mention permits
- at home sale (buyer discovers unpermitted work)
- after inspector visit

---

## 5 whys tree

### branch 1: information access

**symptom**: homeowner doesn't know if permit is required

1. why? → homeowner has not checked the permit requirements
2. why? → homeowner doesn't know where to find permit requirements
3. why? → permit requirements are in legal codes (IRC, local amendments)
4. why? → legal codes are written for professionals, not homeowners
5. why? → **legal code is inaccessible to non-experts**

**root cause**: legal code accessibility

### branch 2: portal confusion

**symptom**: homeowner doesn't know if permit is required

1. why? → homeowner checked the permit portal but didn't find an answer
2. why? → the permit portal shows permits PULLED, not permits REQUIRED
3. why? → the portal is a records system, not a guidance system
4. why? → permit offices are understaffed for guidance calls
5. why? → **permit system designed for compliance record-keep, not citizen guidance**

**root cause**: system design mismatch

### branch 3: work scope ambiguity

**symptom**: homeowner doesn't know if permit is required

1. why? → homeowner describes work differently than code categories
2. why? → code uses technical terms ("service upgrade", "branch circuit")
3. why? → homeowner uses practical terms ("electrical panel", "outlet")
4. why? → no translation layer between homeowner language and code language
5. why? → **vocabulary gap between homeowner and code**

**root cause**: vocabulary translation gap

---

## root cause summary

| layer | issue | is root cause? |
|-------|-------|----------------|
| surface | homeowner doesn't check | no — they want to comply |
| mid | portal shows wrong info | no — portal has different purpose |
| mid | code uses technical terms | contributory |
| deep | no translation layer exists | contributory |
| deep | code inaccessible to non-experts | **yes** |
| deep | system designed for record-keep, not guidance | **yes** |

**two root causes identified**:
1. legal code is inaccessible to non-experts
2. permit system designed for record-keep, not guidance

---

## recommendation

### which layer should we address?

**address root cause #1**: make legal code accessible via automated determination tool

- translate work descriptions into code categories
- look up relevant code sections
- apply exemption criteria
- return determination with verbatim citations

**address root cause #2**: cannot change system design (out of scope)

- permit offices will not redesign their systems
- we can fill the guidance gap they leave

### implementation approach

| root cause | our approach | out of scope |
|------------|--------------|--------------|
| code inaccessibility | permit.check.required skill | rewrite legal codes |
| system design mismatch | permit.search + permit.fetch skills | redesign Accela portal |
| vocabulary gap | work description parser | train permit office staff |

### what this means for the product

the product must:
1. accept homeowner language (not code language)
2. translate to code categories internally
3. look up relevant code sections
4. apply determination logic
5. return answer with verbatim proof

this addresses root cause #1 directly and works around root cause #2.
