# research: product audience

## summary

the primary audience is **homeowners who do their own work** — they need to know if their project requires a permit before they start. secondary audiences include property buyers and real estate professionals who need permit history for due diligence.

---

## who is the audience?

### primary: homeowners who do their own work (DIY-ers)

**profile**:
- own residential property in Indianapolis-Marion County
- plan to do work themselves (electrical, pipe, structural)
- not professional contractors
- may not know permit requirements exist

**context**:
- about to start a project
- heard they "might need a permit"
- unsure how to verify
- want to comply

**constraint**:
- limited time to research
- unfamiliar with legal code
- portal navigation is unclear
- don't want to call the permit office

### secondary: property buyers

**profile**:
- about to purchase residential property
- want to verify past work was permitted
- need evidence for insurance or finance approval

**context**:
- due diligence phase
- inspector flagged unpermitted work
- need to assess risk before close

**constraint**:
- time-sensitive (escrow deadlines)
- need documented proof
- may need to verify multiple properties

### tertiary: contractors (licensed professionals)

**profile**:
- licensed electricians, pipe workers, HVAC techs
- already know permits are required
- need to check permit status for jobs

**context**:
- verify their pulled permit was issued
- check if prior work by others was permitted
- document compliance for clients

**constraint**:
- need quick lookup by permit number
- batch lookups for multiple jobs
- already familiar with portal (but would prefer faster access)

## why do they care?

### homeowners (primary)

| pain | this relieves it |
|------|------------------|
| **fear of fines** | know if permit required before you start |
| **inspection anxiety** | understand what inspectors will check |
| **legal uncertainty** | get verbatim code citations, not guesses |
| **portal confusion** | simple search by address and work type |

### property buyers (secondary)

| pain | this relieves it |
|------|------------------|
| **unpermitted work risk** | verify permit history before purchase |
| **insurance gaps** | document permitted vs unpermitted work |
| **negotiation leverage** | use permit gaps to negotiate price |

### contractors (tertiary)

| pain | this relieves it |
|------|------------------|
| **status check friction** | quick lookup by permit ID |
| **client documentation** | export permit details for invoices |
| **prior work verification** | check if homeowner's past work was permitted |

## what do they care about?

### homeowners (primary)

**success criteria**:
- clear YES/NO answer: "does this work require a permit?"
- verbatim code quotes, not paraphrases
- next steps if permit IS required (where to apply, contact info)
- next steps if permit is NOT required (how to document exemption)

**tradeoffs they'd accept**:
- wait a few seconds for search results
- limited to Indianapolis-Marion County (their jurisdiction)
- caveat that this is not legal advice

**tradeoffs they'd reject**:
- ambiguous answers without evidence
- portal navigation required
- call to permit office required

### property buyers (secondary)

**success criteria**:
- permit history for an address
- permit type, status, date, contractor
- evidence format for lender/insurer

**tradeoffs they'd accept**:
- limited to recent permits (last 10-20 years)
- may not have all historical records

**tradeoffs they'd reject**:
- manual export required
- incomplete records without notice

---

## how will they experience it?

### homeowner journey

1. **trigger**: decides to upgrade electrical panel
2. **question**: "do I need a permit for this?"
3. **action**: runs `permit.check.required --postal 46220 --work "electrical panel upgrade"`
4. **output**: determination with proof chain
   - baseline: IRC R105.1 requires permits for electrical work
   - exemptions: checked R105.2, none apply
   - conclusion: **permit required**
   - next steps: Indianapolis DBNS contact info, portal link
5. **outcome**: knows to apply for permit before work

### friction points

| friction | mitigation |
|----------|-----------|
| doesn't know postal code | use address lookup |
| doesn't know work terminology | provide examples |
| doesn't trust automated answer | show verbatim code quotes |
| needs human confirmation | include permit office contact |

### smooth experience

```
🦅 we have a determination,

🏛️ permit.check.required
   ├─ query
   │  ├─ postal: 46220
   │  └─ work: electrical panel upgrade 100A to 200A
   │
   ├─ determination: required
   │
   ├─ proof
   │  ├─ baseline: IRC R105.1 — electrical systems require permit
   │  ├─ exemptions: none apply — panel upgrade not in R105.2
   │  └─ jurisdiction: Indiana Residential Code 2020
   │
   └─ next steps
      ├─ contact: Indianapolis DBNS permit counter
      ├─ phone: (317) 327-8700
      └─ portal: aca-prod.accela.com/INDY

⚠️ this is not legal advice — consult a licensed attorney
```
