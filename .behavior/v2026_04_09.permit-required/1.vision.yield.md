# vision: permit-required

## the outcome world

### before

- homeowner wonders "do i need a permit for this?"
- contractor says "nah, you don't need one" — but is that true?
- googles, gets conflicting answers from forums, ai, other contractors
- calls permit office, waits on hold, gets verbal answer with no citation
- proceeds uncertain, trusts the contractor's word
- if caught unpermitted: 2x fees, retroactive process, insurance claim rejection

### after

- homeowner runs `rhx permit.check.required --postal 46220 --work "electrical panel upgrade"`
- sees what the code actually says, with exact citations
- proof chain: IRC R105.1 (baseline) → R105.2 (exemptions) → jurisdiction adoption
- now has a second opinion — backed by legal citations, not just "trust me"
- when contractor says "you don't need a permit", homeowner can check for themselves
- knows what questions to ask the permit office or a lawyer if unclear

### the "aha" moment

> "wait, my contractor said i don't need a permit — but the code says i do? now i know what to ask about."

the value clicks when users see verbatim code text, not a contractor's opinion. they still need to verify with the permit office or a lawyer — but now they have a citation to point to, not just someone's word.

**why this matters**: contractors sometimes skip permits to save time or avoid fees. homeowners bear the risk: unpermitted work can void insurance claims, block home sales, or trigger retroactive fees. a second opinion backed by citations protects the homeowner.

## the frame

> we help you find what to ask. we can't tell you what to do.
>
> you still won't know everything. but we'll help find what we can.
>
> it's higher ground to start from. raise the floor.

### this is not legal advice

only barred lawyers can provide legal guidance. our clones produce research, not counsel.

### what this CAN do

| use case | value |
|----------|-------|
| double-check | get a second opinion on what you've been told |
| zoom in | identify which areas need closer attention |
| question prompts | surface what to ask a lawyer about |
| "get it written" | flag situations where documentation matters |
| "ask for a second opinion" | detect when professional review is warranted |

### what this CANNOT do

| use case | why not |
|----------|---------|
| "you're good" | we cannot confirm legal sufficiency |
| "this will hold up in court" | we cannot predict judicial outcomes |
| "you don't need a lawyer" | we cannot assess when professional help is needed |

### the value

contractors sometimes skip permits to save time or cut costs. the homeowner bears the risk:
- unpermitted electrical work → insurance denies fire claim
- unpermitted addition → sale blocked at inspection
- caught years later → 2x fees plus retroactive permit process

the homeowner who runs `permit.check.required` has a second opinion backed by citations. when a contractor says "you don't need a permit", they can check for themselves. that's not legal advice — but it's higher ground than blind trust.

      limit: number

## evaluation

### pros

- **citable**: exact code citations, not "trust me" — users can verify
- **second opinion**: check what a contractor told you against actual code
- **cacheable**: code text is immutable per edition year
- **composable**: same pattern as patenter (search, fetch, propose)
- **pit of success**: proof chain requires both baseline AND exemptions

### cons

- **jurisdiction complexity**: 50 states × thousands of localities
- **code access**: some codes behind paywalls (ICC)
- **permit API access**: Accela requires agency partnership
- **scrape fragility**: web portals change

### edgecases

| edgecase | pit of success |
|----------|----------------|
| postal spans multiple jurisdictions | warn user, list all applicable |
| code not found online | cite source as "not publicly available" |
| work description ambiguous | ask for clarification before research |
| exemption is conditional | include full conditional text |

## what is awkward?

### code access fragmentation

codes live in different places:
- ICC Digital Codes (paywall)
- UpCodes (partial free)
- state fire marshal sites
- municode (local)

no single source of truth.

### jurisdiction identification

given a postal code, which jurisdiction applies?
- city? county? unincorporated?
- some postals span boundaries

need robust postal → jurisdiction mapper.

### structure type determination

IRC vs IBC depends on structure type:
- single-family, two-family → IRC
- multi-family, commercial → IBC

need parcel data lookup to determine structure type from address. user self-report is unreliable (they may not know the legal classification).

### work type taxonomy

users say "panel upgrade" but codes say "replace... electrical... system"

this is an NLP problem:
1. synonym resolution (panel = service = breaker box)
2. intent classification (upgrade vs repair vs new)
3. code terminology match (natural language → legal terms)

**high stakes**: wrong interpretation = wrong research. not the same as legal advice, but still matters — if we pull the wrong code section, the second opinion is useless.

the proof chain now includes `work_interpreted` so users can verify we understood their intent. if confidence is low, we ask for clarification. users should always verify with the permit office or a lawyer before they act.

### permit search varies wildly

- some jurisdictions: full API
- some: web portal only
- some: call the office

can't promise uniform experience.
