## subcommands

| subcommand | verdict | rationale |
|------------|---------|-----------|
| repos | **keep** | discovery usecase: "what repos exist?" |
| files | **question** | could `lines --paths '*'` cover this? |
| lines | **keep** | core usecase: search and read |

### files subcommand analysis

**if deleted**: users would run `lines --paths '*'` to list files, but:
- lines reads content, files lists names — different operations
- `git ls-tree` (files) vs `git show` (lines) — different git commands
- files supports `--words` filter to find files by content pattern
- files returns file tree structure, lines returns line content

**would we add it back?** yes — the usecases are distinct:
- "what files exist?" → files
- "show me this file" → lines

**verdict**: keep files subcommand
